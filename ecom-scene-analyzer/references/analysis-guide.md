## 排版规范

渲染脚本会将 JSON 中的文本字段转换为结构化的可读段落。**生成 JSON 时必须遵循以下三条格式约定**，否则渲染效果会退化为大段无结构文字：

### 1. 语义分段：用 `\n` 分隔

每个独立论点、观察角度、或分析视角占一段，段与段之间用 `\n` 分隔。分段依据是语义——该换论点时就换段，而非机械地每隔几句就断。

### 2. 行内小标题：用 `【标题】` 开头

每段开头用 `【小标题】` 标注这段在讲什么，渲染后会变成行内加粗+冒号的锚点（如「**角度选择的有效性：**正文……」）。小标题应简短（2-8字），概括本段核心论点。

### 3. 关键词标注：用 `<strong>` 包裹

正文中的关键术语、核心结论、重要数据用 `<strong>关键词</strong>` 包裹，渲染后会加粗加深，从正文中跳出来。每段标注 1-3 处，不宜过多。

### 格式示例

```
"attribution": "【角度选择的有效性】左后方3/4视角是展示椅背调节机构的<strong>最优角度</strong>——椅背前后、上下调节的机械结构恰好在背面可见。\n【对比性思考】如果换成正面视角，能看到孩子表情但会<strong>失去所有调节机构的信息量</strong>。\n【品类共识】背面3/4视角在功能展示图中是主流做法，这张图的突破在于将<strong>功能标注叠加到使用场景上</strong>。"
```

此规范适用于 `dimensions` 的 observation 和 attribution、`insights` 的 attribution / meta_thinking / reuse 字段。`overview`、`style_params`、`prompts` 等字段不需要分段处理，也不使用 `<strong>` 标签——这些字段的内容以纯文本输出。

## 分析方法论

### 观察与归因一体化

按五个维度逐一分析。多图时综合分析共同的视觉策略，在观察中标注各图差异。每个维度内部完成"看到了什么→为什么这样做"的完整闭环——先客观记录画面事实，紧接着解释这个视觉选择背后的原因。不将归因拆到独立模块导致事实重复。

分析每个维度时，必须按下方各维度的观察子项清单逐项覆盖——这些子项是分析质量的核心保障，不是可选参考。其中维度五（儿童模特与成长叙事）是儿童学习家居品类中最核心的维度，需要最充分的分析。

### 归因工具箱

归因时，以下六个视角作为**思维工具箱**来审视每个观察点，但只输出真正提供了新理解的归因，不为覆盖框架而强行解释。一条融合了多视角的深度归因，远好于六条各视角的浅层重复：

- **视觉设计原理**：构图法则、动线设计、色彩权重、层级关系
- **心理学机制**：镜像神经元、格式塔原则、色彩心理学、安全感联想
- **营销策略**：场景化卖点、高语境沟通、目标客群锚定、想象成本
- **品类特异性**：家长核心焦虑、体态作为卖点、成长叙事、年龄段锚定
- **文化符号学**：视觉元素的文化编码、阶层认同信号、教育期望投射
- **感官联觉**：视觉→触觉/温度/空间感的联想、材质与感知品质

每个维度的归因中至少尝试一个不那么显而易见的视角（如文化符号学、感官联觉），避免五个维度全部只用"视觉设计原理"这个最直觉的视角。

### 品类共识对照

每个维度的归因末尾，用一句话明确这个视觉选择在品类坐标系中的位置——是遵循主流共识、有意突破、还是偏离最佳实践。对照依据是 `references/category-knowledge.md` 中的品类规律。突破不等于错误，偏离也不等于失败，但设计师需要知道。

### 对比性思考

分析"为什么这样做"时，同时思考"如果不这样做/换一种做法会怎样"。这是区分表面描述和真正洞察的关键。每个维度的归因中应包含至少一处对比性分析（"如果换成X，会失去Y/得到Z"）。

### 区分类型再分析

场景图追求代入感和沉浸感，主图追求注意力抓取和能量感。在总体印象中先判断图片类型，后续分析视角对应调整。

## 核心洞察

五维分析完成后，提炼 2-4 条核心洞察。这是整份分析中最高价值的部分。

**筛选标准**：只有能上升到元思考层面的发现，才值得写成核心洞察。停留在归因层面的分析，在五维分析中完成即可，不必重复进入此模块。宁可只写两条有深度的洞察，也不要四条停在归因层的重复。

每条核心洞察必须完成三层递进：

1. **归因**——这个视觉选择为什么有效？（含对比性思考和品类共识定位）
2. **元思考**——这背后折射的底层规律是什么？跳出这张图，这个规律在更广的设计语境中意味着什么？
3. **复用**——基于这个规律，设计师在下一个项目中具体可以怎么做？或者可以探索什么方向？

**元思考的定义**：不是解释"这张图为什么这么做"（那是归因），而是追问"这张图的做法折射出的底层逻辑是什么"。好的元思考能让设计师获得一个可迁移到其他品类、其他场景的认知框架，而非仅限于儿童学习椅的具体经验。在适用的情况下，还可以追问：这个视觉选择精准命中了家长的哪种焦虑或期待？这种命中是怎么被设计出来的？

**自检**：写完每条洞察后问自己——"如果去掉这张图的具体信息，这条洞察是否仍然对设计师有指导价值？"如果答案是否，它还停留在归因层，应该留在五维分析中而非升格为核心洞察。

**示例**——

> **归因**：场景图中孩子脚自然踩在踏板上，但画面中没有任何标注或箭头指向踏板，脚踏板的卖点完全通过"正在被使用"来传达。相比功能标注图，这种方式让家长在潜意识中完成了"这个踏板有用→我的孩子也需要"的认知闭环，说服力更强。
>
> **元思考**：产品功能在场景图中的最佳表达方式是"被使用"而非"被说明"——这背后是"体验性认知"与"分析性认知"的差异。当家长看到孩子的脚踩在踏板上，他们激活的是体验性认知（"我的孩子坐上去也会这样"），这比看到一个标注箭头激活的分析性认知（"这个踏板可以调节高度"）更接近购买决策时的心理状态。这条规律不限于踏板——任何产品功能，只要能被"正在发生"的画面承载，就不应该退回到图文标注。
>
> **复用**：拍摄场景图时，列出产品的三个核心卖点，逐一检查每个卖点是否可以通过人物的自然使用状态来展示，而非标注。只有确实无法通过使用状态展示的功能（如内部结构、材质成分），才留给独立的功能详解图。

## 风格参数提取

将分析浓缩为一组可传递的风格参数。这组参数既供设计师审阅理解图片的视觉策略，也作为提示词生成的组装素材。

参数以自然语言描述为主，但**机位角度需给出数值估算**，便于精确复现。光线和色调字段中**补充实操参数**（色温范围、主色数量、拍摄时段建议等），使参数不仅能喂给生成模型，也能直接用于拍摄现场。

每个字段除了设计师可读的视觉描述外，还应包含 AI 生成工具执行所需的精确信息——角度数值、间距描述、色彩策略意图等。这些信息是提示词组装的原料，如果在参数阶段丢失，提示词就不得不用"自然""舒适"等生成工具无法解析的模糊词来填补。

以下模板中，括号内为对应的 JSON 字段名（`style_params` 下）：

```
场景模板命名（template_name）：[一个短语定位场景类型，如"窗边自然光学习角"/"温馨居家儿童房角落"]

- 机位（camera）：[仰角/俯角约X°，水平偏转约X°（正面/侧面/背面/3/4），
  拍摄距离感（特写/中景/全景），
  透视方向（灭点位置、地面/桌面线条汇聚方向），
  镜头焦段感受（广角变形感/中焦压缩感）]

- 光线（lighting）：[主光方向与质感，反差感受，阴影软硬，整体氛围。
  光线在产品上的高光位置和阴影位置，
  地面投影的方向和强度级别（无/极淡/明显），
  人物和产品的光影方向是否一致。
  实操参考：推荐拍摄时段、光源方案（如"上午9-11点侧窗自然光+白纱柔化"）]

- 色调（color_tone）：[色温倾向，饱和度感受，暗部处理方式，整体情绪。
  产品色与环境色的区隔方式（如"产品冷白 vs 环境暖米"），
  服装色在画面色彩关系中的视觉权重定位
  （为什么选这个颜色/图案，与产品色和环境色的关系）。
  实操参考：大致色温范围（如约4000-4500K）、
  主色数量与占比控制（如"≤3种主色，白+木+浅灰占比约70%"）]

- 人椅空间关系（human_chair_space）：[人椅比例（头顶与椅背的相对关系），
  各接触面的间距或角度（臀部-坐垫、背-椅背、脚-踏板、
  手臂-扶手/桌面），用具体数值或可感知的对比描述
  （如"背部与椅背留约1-2cm间隙"而非"自然贴合"），
  身体朝向的不对称性（哪侧前/后，重心偏向），
  四肢的可见性（哪些被遮挡、哪些露出、各在什么位置），
  关节角度]

- 人物（分两层）：
  · 表象（character_surface）：服装款式与颜色、发型、面部朝向、估算年龄段——客观外观事实。
    发型对产品轮廓遮挡的影响（需要露出哪些区域）
  · 本质（character_essence）：这个人物整体向家长传递的气质信号——不是外观描述，
    而是家长视角的感受。造型选择服务于哪个形象原型？
    服装色彩的视觉权重策略
    （为什么选这个颜色/图案而非其他，替换时需维持什么原则）

- 成长叙事（growth_narrative）：[活动类型，功能展示方式（show don't tell 的程度），成长感强弱]

- 场景道具（props）：[按功能分类标注每件道具的角色，如：
  · 草编花篓——自然系收纳·氛围道具
  · 绿植——健康联想·感官道具
  · 书架——生活场景·背景道具]

- 构图（composition）：[主体位置，动线走向，景深层次，留白位置与比例。
  画幅比例（宽高比）、主体组合（人+椅+桌）在画面中的面积占比和位置、
  垂直分区参数（顶部留白比例、底部留白比例、桌面横切线位置）、
  产品完整度要求（椅子是否必须完整入画）、桌面入画方向和比例。
  前景遮挡物的空间功能（是否部分遮挡主体、遮挡比例约多少、
  制造什么空间关系），
  留白的功能意图（文案区/呼吸感/裁切安全区）]

- 关键约束（key_constraints）：从分析中提炼 3-5 条不可妥协的生成红线，
  每条用"必须X——因为如果不X会Y"的格式。
  Y 的来源是五维分析中的对比性思考——
  凡是分析中出现"如果换成X会失去Y"的判断，
  说明当前选择是有意的控制点，应被提炼为关键约束。
```

`style_params` 中所有字段以纯文本输出，不使用排版规范中的分段和 `<strong>` 标签。

## JSON 字段定义（分析部分）

### meta
报告头副标题信息，脚本会用 `·` 连接显示。
- `category`：产品品类（如"儿童学习椅"）
- `description`：图片描述（如"背面坐姿场景图"）
- `usage`：用途（如"详情页用"）

### images
图片文件路径数组（如 `["/mnt/user-data/uploads/xxx.png"]`）。脚本会自动转为 base64 内嵌。

### overview
- `type`：场景图（详情页）或 主图（搜索列表）
- `consensus`：品类共识定位——遵循主流 / 有意突破 / 偏离最佳实践 + 一句话说明
- `strategy`：一句话概括核心视觉策略

---

### dimensions

五个维度的 key 固定为 `angle`、`light`、`color`、`comp`、`model`。每个维度包含 `observation`（观察）和 `attribution`（归因），均遵循上方排版规范（`\n` 分段 + `【小标题】` + `<strong>` 关键词）。

**分析时严格按照以下各维度的观察子项逐项覆盖，不要跳过。**

#### angle（拍摄角度）

**observation 应覆盖：**
1. 机位高度：相对于产品和人物的位置（平视/俯拍/仰拍），估算仰角/俯角角度
2. 水平方向：正面/侧面/背面/3/4视角，估算偏转角度
3. 信息密度：这个角度同时展示了产品的哪些信息（椅背造型、扶手、脚踏板、座垫、调节机构等）
4. 人物与产品的主次关系：角度如何处理两者的视觉权重分配

**attribution 应回答：**
- 这个角度选择为什么有效？
- 对比性思考：如果换成另一个角度（如正面/俯拍/侧面）会失去什么、得到什么？
- 品类共识对照：这个角度选择是遵循/突破/偏离品类主流做法？

#### light（光线与影调）

**observation 应覆盖：**
1. 主光方向与光源类型（窗光/棚灯/混合）
2. 光线质感：硬光还是柔光，反差高低
3. 阴影状态：软硬程度、方向，是否有底部投影锚定真实感
4. 人物身上的光：光线如何落在儿童身上，是否有硬阴影

**attribution 应回答：**
- 这种光线处理在儿童学习家居品类中为什么合理？
- 对比性思考：如果换成硬光/高反差/暗调会发生什么？
- 品类共识对照：是否遵循品类主流（大面积柔光、低反差、高调）？

#### color（色调与质感）

**observation 应覆盖：**
1. 整体色温倾向与饱和度水平
2. 色彩关系：产品颜色、服装颜色、环境颜色三者之间的关系——统一还是对比，谁在视觉上跳出来
3. 暗部处理：画面最暗处是纯黑还是被提亮了
4. 材质呈现：产品表面的哑光/亮光/柔光质感

**attribution 应回答：**
- 这组色彩策略对消费者感知产生什么影响？
- 服装色彩与环境的关系是有意为之吗，目的是什么？
- 品类共识对照：是否遵循品类主流（偏暖调、低饱和度、暗部提亮）？

#### comp（构图与视觉动线）

**observation 应覆盖：**
1. 主体位置（三分法/居中/偏移）
2. 视觉动线：观者视线的移动路径
3. 景深层次：前景/中景/背景分别是什么，清晰度如何分层
4. 留白面积与位置
5. 道具：有哪些，在什么位置，起什么作用
6. 画幅比例：宽高比（如约3:4竖版、4:3横版、1:1方形）
7. 主体占比：「人+椅」组合在画面中的高度占比和宽度占比，桌面入画后「人+椅+桌」整体占画面的面积比例
8. 垂直分区：头顶到画面上沿的留白比例、椅脚/踏板到画面下沿的留白比例、桌面边缘在画面中的垂直位置（从上沿算起的百分比，如"桌面横切线位于画面约45%处"）——桌面这条水平线将画面分为上方「人物叙事区」和下方「产品结构展示区」，两区面积比直接影响信息权重分配
9. 产品完整度：椅子是否从椅背顶端到椅脚底部完整入画（学习椅品类中近乎硬性要求——A字腿和踏板都是功能卖点，裁切任何一端都丢信息）
10. 桌面裁切方式：桌面从哪个方向切入画面（右侧/左侧/顶部）、入画宽度占画面比例多少——桌面只露局部是为了暗示空间延伸感，同时不让桌面分散椅子（主产品）的注意力

**attribution 应回答：**
- 这个构图如何引导消费者的注意力？
- 道具选择传递了什么信号？（文化符号学视角：道具的阶层认同/生活方式编码）
- 品类共识对照：构图和道具使用是遵循/突破/偏离品类主流做法？

#### model（儿童模特与成长叙事）——最核心的维度

**observation 应覆盖以下六个话题，输出时用 `【小标题】` 格式（与其他维度统一），不要使用编号分项：**
- 体态与坐姿（最重要）：脊柱状态、脚与踏板的关系、手臂与桌面/扶手的位置、背部与椅背的贴合程度
- 服装与发型：颜色、款式、风格，与产品和环境的配合关系；发型是否遮挡产品轮廓
- 面部处理：面部朝向（正面/侧面/背面）、是否看镜头、表情
- 活动选择：孩子在做什么（写字/阅读/画画/看平板等）
- 安全感：人物与产品的身体比例关系，是否让人感到安全合适
- 成长信号：画面中是否有暗示"产品可伴随成长"的元素（可调节机构的可见性等）

每项用 `<strong>` 标注关键发现。

**attribution 应回答：**
- 体态和活动选择如何回应家长的核心焦虑？
- 面部朝向的选择对家长的心理代入产生了什么影响？
- 对比性思考：如果换成正面看镜头/做其他活动，效果会有什么不同？
- 品类共识对照：人物处理是遵循/突破/偏离品类主流做法？
- **气质信号收束**：综合所有观察，用一句家长视角的话概括人物整体传递的气质信号（如"这就是我希望孩子写作业时的样子"）
